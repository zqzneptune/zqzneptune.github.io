Chapter 8: Alignment Mechanics and Mapping Formats (SAM/BAM)

Published

June 5, 2026

Modified

July 5, 2026

Mapping Billions of Reads Perfectly is Computationally Impossible

If you attempt to align a single 150 bp sequencing read to the 3.2 billion bp human genome using exact mathematical models, you will compute billions of alignment combinations. If you multiply this by 300 million reads from a single sequencing lane, your pipeline will run for decades. Bioinformatics is the science of engineered heuristics—sacrificing mathematical perfection for speed. Every aligner is wrong; some are useful.

Core Concepts

Once your sequencer outputs raw FASTQ files, your next task is to determine where each read originated in the reference genome. This step is called sequence alignment (or mapping).

To choose the right aligner and understand its output, you must understand the mathematical limits of alignment, the heuristics used to bypass them, and the resulting file formats.

The Computational Wall: Dynamic Programming

The mathematically “perfect” way to align two sequences is through Dynamic Programming (DP), specifically the Needleman-Wunsch (for global alignment) and Smith-Waterman (for local alignment) algorithms. These algorithms build a massive scoring matrix where every cell represents a comparison between a base in the read and a base in the genome.

For a read of length \(L\) and a genome of length \(G\), the complexity of dynamic programming is: \[\text{Time Complexity} = O(L \times G)\]

For a single 150 bp read against the 3.2 billion bp human genome, the matrix contains \(150 \times 3.2 \times 10^9 = 480 \text{ billion cells}\). To calculate this for millions of reads is computationally impossible. Therefore, aligners do not search the genome using dynamic programming from scratch.

The Heuristic Escape: Seed-and-Extend

To bypass this computational wall, modern aligners use the Seed-and-Extend heuristic:

  1. Seeding: The aligner identifies short, exact matches (e.g., 20 bp) between the read and the genome. These exact matches are called “seeds”. To do this quickly, the aligner uses a pre-built index of the reference genome.

  2. Extension: Once the aligner finds a seed location, it halts the genome-wide search and performs local dynamic programming (Smith-Waterman) in the immediate flanking region around the seed to align the rest of the read, allowing for mismatches and gaps (indels).

Classifying Aligner Methods: Choosing the Right Tool

Different biological assays require different alignment behaviors. Aligners are classified into three primary categories based on the prior assumptions they make:

1. DNA Aligners (BWA-MEM & Bowtie2)

  • The Assumption: Reads align contiguously to the genome with few gaps or mismatches.

  • The Technology: These tools index the genome using the Burrows-Wheeler Transform (BWT) and the FM-index, which compresses the 3 Gb human genome into a ~2 GB index that fits in RAM, allowing exact seed searches in linear time.

  • Usage: BWA-MEM is the gold standard for variant calling (WGS/WES); Bowtie2 is preferred for short reads with minimal gaps (e.g., ChIP-seq or metagenomic profiling).

  • The BWA-MEM Parameter Choices: In production genomic pipelines, running BWA-MEM with default parameters is a rookie mistake. I recommend using two crucial flags:

    1. -M: This flag marks shorter split hits as secondary (rather than supplementary). This is a critical prerequisite for downstream duplicate marking tools (like Picard MarkDuplicates or GATK). If you omit the -M flag, these tools will fail to recognize and eliminate PCR duplicates from split-aligned reads, artificially inflating your depth and causing false somatic variant calls.
    2. -K <integer>: In multi-threaded execution, BWA-MEM processes reads in chunks. By default, the chunk size changes dynamically based on thread scheduling, leading to slightly different alignment outputs depending on the thread count (non-deterministic runs). Specifying a fixed chunk size (e.g., -K 100000000) forces BWA-MEM to process identical chunks regardless of threading, guaranteeing 100% computational reproducibility across runs.

2. RNA-Seq Splice-Aware Aligners (STAR)

  • The Assumption: Reads originate from spliced messenger RNA transcripts. Because splicing removes introns, a single read can cross an exon-exon junction, meaning one half of the read maps to Exon 1, and the other half maps to Exon 2 (separated by thousands of base pairs in DNA space).

  • The Technology: STAR identifies Maximal Mappable Seeds on either side of the junction and aligns them separately, bridging the intron gap and looking for canonical splice site signatures (e.g., GT-AG).

  • Usage: Standard transcriptome profiling (RNA-seq).

3. Long-Read Aligners (Minimap2)

  • The Assumption: Reads are long (10 kb to 1 Mb) but have high error rates (1% to 5%).

  • The Technology: Minimap2 uses minimizers—a subset of representative k-mers—to index the reference and seed alignments. This indexing strategy is highly tolerant of mismatches and gaps, allowing noisy long reads to find their correct genomic coordinates.

  • Usage: Long-read DNA/RNA alignment (PacBio/Oxford Nanopore).

The Mapping Output: SAM/BAM Format

The output of an aligner is a SAM (Sequence Alignment/Map) file (or its binary, compressed version, the BAM file). A BAM file is the foundational input for almost all downstream assays (ChIP-seq peak calling, RNA-seq quantification, and Variant Calling).

The Anatomy of a BAM File

A BAM file consists of a header section followed by tab-delimited alignment records.

The Header

Always starts with @ and contains crucial metadata:

  • @HD: Format version and sorting order (e.g., coordinate sorted).

  • @SQ: Reference sequence dictionary (chromosome names and lengths).

  • @RG: Read Groups, mapping which reads belong to which biological sample (essential for batch identification).

The Alignment Records

Every read has a record with 11 mandatory fields:

SRR1234567.1    83    chr1    10045    30    150M    *    0    0    GATC...    F:FF...

  1. QNAME (Query Name): The unique name of the read from the FASTQ header.

  2. FLAG (Bitwise Flag): A single integer that encodes multiple mapping properties (see below).

  3. RNAME (Reference Name): The chromosome name where the read aligned.

  4. POS (Leftmost Position): The 1-based start coordinate of the alignment.

  5. MAPQ (Mapping Quality): The probability that the read is placed at the wrong locus (see below).

  6. CIGAR (CIGAR String): A string summarizing the match, mismatch, insertion, and deletion operations (e.g., 150M = 150 matches; 30M2D120M = 30 matches, 2 deletions in reference, 120 matches).

  7. RNEXT: Name of the reference for the mate read (if paired-end).

  8. PNEXT: Leftmost position of the mate read.

  9. TLEN (Template Length): Estimated size of the sequenced DNA fragment.

  10. SEQ: The read sequence.

  11. QUAL: The quality string.

Understanding MAPQ (Mapping Quality)

The MAPQ score represents the probability (\(P_m\)) that the aligner has placed the read at the wrong location: \[\text{MAPQ} = -10 \log_{10}(P_m)\]

  • MAPQ 30: A 1-in-1,000 chance that the read is mismapped (\(99.9\%\) accuracy).

  • MAPQ 0: The read maps equally well to multiple locations in the genome (e.g., repetitive regions). The aligner picked one position at random.

[!WARNING] Filter MAPQ 0 in Variant Calling: If you do not filter out MAPQ 0 reads before calling variants, your pipeline will identify thousands of false mutations that are actually just misplaced reads from repeat regions. I recommend filtering for \(\text{MAPQ} \ge 20\) or \(\text{MAPQ} \ge 30\) in germline pipelines.

Decoding the Bitwise FLAG

The FLAG field is a bitwise combination of 12 distinct binary flags. Each flag represents a boolean question. Rather than writing 12 columns, the aligner sums the binary values of the active flags:

Bit Decimal Value Meaning
1 1 Read is paired-end
2 2 Read mapped in a proper pair (correct distance and orientation)
3 4 Read is unmapped
5 16 Read mapped to the reverse strand
7 64 This is Read 1 of the pair
8 128 This is Read 2 of the pair

For example, a FLAG value of 83 represents: \[83 = 1 + 2 + 16 + 64\] This means: the read is paired (1), mapped in a proper pair (2), mapped to the reverse strand (16), and is Read 1 of the pair (64). You do not need to calculate this manually; tools like samtools use these flags to filter reads (e.g., samtools view -f 2 extracts only properly paired reads).

Biological Interpretation

When analyzing BAM files, you must be alert to alignment bias—the tendency of aligners to preferentially map reads that match the reference genome perfectly over reads that carry non-reference variants.

Soft-Clipping vs. Hard-Clipping

When the end of a read contains sequence that does not match the reference (e.g., unremoved adapters or highly mutated regions), the aligner will clip the ends.

  • Soft-Clipping (S in CIGAR): The aligner keeps the clipped bases in the BAM record but ignores them during alignment scoring (e.g., 10S140M).

  • Hard-Clipping (H in CIGAR): The aligner removes the clipped bases from the BAM record entirely to save space.

If you see high rates of soft-clipping in your alignments, it is a diagnostic clue: either your adapter trimming step failed, or your sample has a structural insertion that differs significantly from the reference genome.

Summary and Required Reading

  1. Genome-wide exact alignment is too slow: Aligners use indexing and seed-and-extend heuristics to achieve speed.

  2. Select the correct aligner: Use DNA aligners for contiguous mapping, STAR for RNA splicing, and Minimap2 for long reads.

  3. MAPQ measures mapping confidence: MAPQ 0 means the read aligns to multiple repeat copies in the genome.

  4. BAM is the universal format: A binary alignment format representing the raw input for all variant calling, peak calling, and transcriptomics pipelines.

Required Reading

  • Li et al.: “The Sequence Alignment/Map format and SAMtools” (Bioinformatics, 2009). The original specification describing the design of SAM/BAM files.

Johnson’s Rule: A BAM file is the foundation of modern sequencing analysis. If your MAPQ scores are low, your reads are mapping ambiguously. Do not proceed to variant or peak calling until you audit your mapping quality.
